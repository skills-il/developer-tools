### שלב 2: הגדרת RTL ברמת המצגת

כל מצגת בעברית חייבת לכלול הגדרת כיוון RTL הן ברמת השקופית והן ברמת מסגרת הטקסט. ספריית python-pptx אינה חושפת את מאפייני ה-RTL ישירות, ולכן יש לעבוד עם ה-XML ישירות.

**הגדרת כיוון פסקה מימין לשמאל:**

```python
def set_paragraph_rtl(paragraph):
    """הגדרת כיוון טקסט מימין לשמאל עבור פסקה."""
    pPr = paragraph._p.get_or_add_pPr()
    pPr.set('algn', 'r')  # יישור ימני
    pPr.set('rtl', '1')   # הפעלת כיוון RTL
```

**הגדרת כיוון מסגרת טקסט:**

```python
def set_textframe_rtl(text_frame):
    """הגדרת מסגרת טקסט שלמה ל-RTL."""
    txBody = text_frame._txBody
    bodyPr = txBody.find(qn('a:bodyPr'))
    if bodyPr is None:
        bodyPr = txBody.makeelement(qn('a:bodyPr'), {})
        txBody.insert(0, bodyPr)
    bodyPr.set('rtlCol', '1')
    for paragraph in text_frame.paragraphs:
        set_paragraph_rtl(paragraph)
```

### שלב 4: טיפול בטקסט דו-כיווני (Bidi)

מצגות בעברית משלבות לעיתים קרובות טקסט בעברית ובאנגלית, מספרים ותווים מיוחדים.

**מספרים בטקסט עברי:** מספרים תמיד מוצגים משמאל לימין (LTR), גם בתוך פסקאות RTL. PowerPoint מטפל בכך אוטומטית כאשר ה-RTL מוגדר כראוי.

**תוכן מעורב עברית-אנגלית:**
1. הגדרת הפסקה ל-RTL (הכיוון הדומיננטי)
2. שימוש ב-runs נפרדים לקטעי טקסט בעברית ובאנגלית
3. החלת הגופן המתאים לכל run בנפרד

### שלב 5: מוסכמות מצגות עסקיות ישראליות

**מבנה מצגת משקיעים (סטנדרט VC ישראלי):**
1. שקופית פתיחה: שם חברה (עברית + אנגלית), לוגו, סלוגן
2. בעיה: נקודת הכאב, נתוני שוק ישראלי
3. פתרון: מה אתם בונים, צילומי מסך של המוצר
4. גודל שוק: TAM/SAM/SOM עם נתונים ישראליים + גלובליים (בשקלים ודולרים)
5. מודל עסקי: מקורות הכנסה, תמחור בשקלים
6. משיכה: מדדים, גרפי צמיחה, לוגואים של לקוחות ישראליים
7. צוות: מייסדים עם רקע צה"לי/8200 (אם רלוונטי), תארים אקדמיים
8. תחרות: מטריצת תחרות
9. תחזיות פיננסיות: תחזית ל-3-5 שנים בשקלים, אופציונלית בדולרים
10. בקשה: סכום הגיוס בשקלים/דולרים, שימוש בכספים

**מוסכמות עיצוב למצגות ישראליות:**
- מטבע: סימן השקל לפני המספר
- תאריכים: DD/MM/YYYY (תקן ישראלי), או שמות חודשים עבריים
- מספרי טלפון: 05X-XXX-XXXX או 972-5X-XXX-XXXX+
- אחוזים: מספר ואחריו סימן %
- מספרים גדולים: פסיק כמפריד אלפים (1,000,000)

## פתרון בעיות

- **בעיה**: טקסט עברי מופיע הפוך או משובש בקובץ PPTX
  **פתרון**: יש לוודא שגם `rtl='1'` בפסקה וגם `rtlCol='1'` ב-bodyPr מוגדרים. שניהם נדרשים לתצוגה נכונה.

- **בעיה**: מספרים מוצגים בסדר שגוי בתוך משפטים בעברית
  **פתרון**: מספרים הם טבעית LTR. אם הם מופיעים שגוי, יש לוודא שדגל ה-RTL של הפסקה מוגדר נכון. אין להפוך מחרוזות מספרים ידנית.

- **בעיה**: גופנים חוזרים ל-Times New Roman או Arial עבור תווים עבריים
  **פתרון**: יש להגדיר את גופן ה-Complex Script באמצעות אלמנט `a:cs`. ללא הגדרה זו, PowerPoint משתמש בגופן הלטיני שעשוי שלא לתמוך בגליפים עבריים.

- **בעיה**: נקודות תבליט מיושרות לשמאל במקום לימין
  **פתרון**: יש להגדיר גם יישור פסקה לימין (`algn='r'`) וגם כיוון RTL (`rtl='1'`). כמו כן, יש לבדוק ש-`rtlCol='1'` מוגדר ב-bodyPr של מסגרת הטקסט.

- **בעיה**: טקסט מעורב עברית/אנגלית נשבר שלא כראוי
  **פתרון**: יש להשתמש ב-runs נפרדים לקטעי טקסט בעברית ובאנגלית באותה פסקה. יש להחיל את הגופן המתאים לכל run בנפרד.
