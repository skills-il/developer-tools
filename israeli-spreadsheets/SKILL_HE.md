### שלב 3: מימוש פונקציות חישוב מס ישראלי

**חישוב מס הכנסה:**
חישוב מס הכנסה ישראלי מתבצע במדרגות (מס שולי). כל מדרגה חלה רק על החלק של ההכנסה שנמצא בטווח שלה. לאחר חישוב המס הגולמי, מפחיתים את נקודות הזיכוי.

**חישובי מע"מ:**
- הוספת מע"מ: סכום * 1.17
- חילוץ מע"מ מסכום כולל: סכום / 1.17 = סכום לפני מע"מ
- חישוב סכום מע"מ: סכום * 0.17

דיוק: תמיד להשתמש ב-`Decimal` של Python ולא ב-`float` לחישובים כספיים. עיגול לפי `ROUND_HALF_UP` בהתאם לתקני חשבונאות ישראליים.

### שלב 4: יצירת חוברות עבודה מעוצבות

**הגדרת גיליון עבודה ישראלי:**

```python
def create_israeli_workbook(title='Sheet1'):
    """יצירת חוברת עבודה עם הגדרות ישראליות."""
    wb = Workbook()
    ws = wb.active
    ws.title = title
    ws.sheet_view.rightToLeft = True  # כיוון RTL
    return wb, ws
```

**עיצוב שורת כותרת:**
- גופן: Heebo, גודל 12, מודגש, לבן
- רקע: כחול כהה (#1F4E79)
- יישור: ימין, אנכי מרכז, גלישת טקסט
- גבולות: דקים מכל הצדדים

### שלב 5: תבניות פיננסיות ישראליות נפוצות

**חשבונית מס (חשבונית מס):**
מבנה חשבונית מס ישראלית:
1. כותרת: "חשבונית מס"
2. פרטי העסק: שם, ח.פ./ע.מ., כתובת, טלפון
3. פרטי חשבונית: מספר, תאריך, תנאי תשלום
4. פרטי לקוח: שם, ח.פ./ע.מ., כתובת
5. טבלת פריטים: מס"ד, תיאור, כמות, מחיר יחידה, סה"כ
6. סיכום: סכום לפני מע"מ, מע"מ (17%), סה"כ לתשלום

**תלוש משכורת:**
מבנה תלוש משכורת ישראלי:
1. כותרת: "תלוש משכורת"
2. פרטי עובד/ת: שם, ת.ז., חודש שכר, תאריך תחילת עבודה
3. תשלומים: שכר בסיס, שעות נוספות, בונוס, החזר הוצאות
4. ניכויים: מס הכנסה, ביטוח לאומי, מס בריאות, פנסיה (עובד), קרן השתלמות (עובד)
5. שכר נטו

**מחשבון ארנונה:**
שיעורי ארנונה משתנים לפי עיר וסוג נכס. שיעורים קירוביים למגורים לכל מ"ר לתקופה דו-חודשית (2025):
- תל אביב: 55.80 שקלים
- ירושלים: 40.50 שקלים
- חיפה: 33.20 שקלים
- באר שבע: 27.90 שקלים
- נתניה: 43.10 שקלים

חישוב: תעריף * שטח (מ"ר) = תשלום דו-חודשי. שנתי = דו-חודשי * 6.

**פלט**: קובץ XLSX עם תאים להזנת הכנסה שנתית, טבלת פירוט מדרגות מס 2025 (10%, 14%, 20%, 31%, 35%, 47%, 50%), חישוב מס לכל מדרגה, ניכוי נקודות זיכוי, ביטוח לאומי, מס בריאות, והכנסה נטו סופית.

## פתרון בעיות

- **בעיה**: הגיליון מוצג משמאל לימין במקום מימין לשמאל
  **פתרון**: יש להגדיר `ws.sheet_view.rightToLeft = True` בגיליון העבודה. הגדרה זו שולטת בכיוון הוויזואלי של עמודות וגלילה.

- **בעיה**: סימן השקל אינו מוצג נכון
  **פתרון**: יש להשתמש בסימן השקל של Unicode במחרוזת העיצוב. יש לוודא שהגופן תומך בסימן השקל. יש לנסות את הפורמט `'#,##0.00 "₪"'`.

- **בעיה**: חישובי מדרגות מס מראים תוצאות שגויות
  **פתרון**: יש לוודא שמדרגות המס מוחלות באופן פרוגרסיבי (שיעורי מס שוליים, לא אחידים). כל מדרגה חלה רק על חלק ההכנסה שנמצא בטווח של אותה מדרגה.

- **בעיה**: שגיאות דיוק עשרוני בחישובים פיננסיים
  **פתרון**: יש להשתמש בטיפוס `Decimal` של Python במקום `float` לכל החישובים הכספיים. יש להגדיר עיגול ל-`ROUND_HALF_UP` בהתאם לתקני חשבונאות ישראליים.

- **בעיה**: כותרות עמודות בעברית מוצגות כטקסט משובש
  **פתרון**: openpyxl מטפלת ב-Unicode באופן מובנה. יש לוודא שקובץ Python שמור כ-UTF-8. אם הכותרות עדיין מופיעות שגוי, יש להגדיר במפורש גופן שתומך בעברית (כגון Heebo, David).
