## הוראות

**הבהרה חשובה:** כלי זה מספק הנחיות כלליות בלבד. דיני חברות, מיסוי ורגולציית ניירות ערך בישראל הם מורכבים. תמיד התייעצו עם עורך דין ורואה חשבון ישראליים מורשים לפני קבלת החלטות מחייבות.

### שלב 1: זיהוי שלב הסטארטאפ

| שלב | צרכים טיפוסיים | פעולות מרכזיות |
|-----|----------------|----------------|
| רעיון / Pre-seed | הקמת חברה, מימון ראשוני | רישום בע"מ, הגשה לתנופה |
| Seed | השקעה ראשונה, בניית צוות | הסכם SAFE / שטר המיר, תוכנית 102 |

### שלב 2: הקמת חברה

**רישום חברה בע"מ ישראלית:**

```
שלבי רישום:

1. בחירת שם חברה
   - בדיקת זמינות: ica.justice.gov.il
   - חייב להיות ייחודי, בעברית או באנגלית
   - סיומת: בע"מ

2. הכנת תקנון
   - תבנית סטנדרטית זמינה ברשם החברות
   - התאמה: סוגי מניות, הרכב דירקטוריון, מגבלות העברה
   - מומלץ: שימוש בתקנון שנוסח על ידי עורך דין לסטארטאפים

3. מינוי דירקטורים ראשונים
   - מינימום: דירקטור אחד
   - תושבות ישראלית אינה נדרשת (אך מעשית לפתיחת חשבון בנק)
   - ת.ז. של הדירקטור או דרכון לדירקטורים זרים

4. רישום מקוון
   - פורטל: ica.justice.gov.il
   - אגרה: כ-2,600 ש"ח
   - לוח זמנים: 3-7 ימי עסקים
   - מסמכים: תקנון, מינוי דירקטורים, כתובת רשומה

5. לאחר הרישום
   - פתיחת חשבון בנק (לאומי, הפועלים, דיסקונט)
   - רישום במס הכנסה בפקיד השומה המקומי
   - רישום למע"מ אם ההכנסות הצפויות חורגות מהסף
   - רישום בביטוח לאומי כמעסיק
```

**דוגמה לחלוקת מניות מייסדים:**
```python
def calculate_founder_allocation(founders, vesting_months=48, cliff_months=12):
    """Calculate founder share allocation with vesting."""
    total_shares = 10_000_000  # Common Israeli startup starting point
    allocation = []

    for founder in founders:
        shares = int(total_shares * founder["percentage"] / 100)
        allocation.append({
            "name": founder["name"],
            "shares": shares,
            "percentage": founder["percentage"],
            "vesting_months": vesting_months,
            "cliff_months": cliff_months,
            "shares_at_cliff": shares * cliff_months // vesting_months,
            "monthly_vesting_after_cliff": shares // vesting_months,
            "share_class": "Ordinary",
        })

    return {
        "total_authorized": total_shares,
        "allocations": allocation,
        "unallocated": total_shares - sum(a["shares"] for a in allocation),
        "note": "Reserve 10-15% for employee option pool (ESOP)"
    }
```

### שלב 5: הקמת תוכנית 102

**הקמת תוכנית אופציות לעובדים:**

```
מסלול רווח הון 102 -- שלבי הקמה:

1. ניסוח תוכנית ESOP
   - שכרו עורך דין ישראלי לדיני עבודה/מיסוי
   - הגדירו: גודל מאגר, לוח הבשלה, מחיר מימוש, נאמן

2. בחירת נאמן מאושר ע"י רשות המסים
   - נאמנים עיקריים: נאמנות בנק לאומי, נאמנות בנק הפועלים,
     IBI נאמנות, אלטשולר שחם
   - עלות: דמי הקמה + דמי שנתיים לכל משתתף

3. הגשת התוכנית לרשות המסים
   - הגשת מסמך התוכנית לפקיד השומה המקומי
   - תקופת התנגדות של 30 יום (רשות המסים יכולה להתנגד או לשנות)
   - התוכנית נכנסת לתוקף לאחר 30 יום אם אין התנגדות

4. הענקת אופציות לעובדים
   - החלטת דירקטוריון לכל הענקה
   - הסכם אופציות חתום על ידי העובד
   - הנאמן מקבל הודעה ומנהל את ההפקדה

5. הבשלה ותקופת החזקה
   - סטנדרטי: הבשלה ב-4 שנים, cliff של שנה
   - תקופת חסימה של 24 חודשים ממועד ההענקה (למסלול רווח הון)
   - מניות מוחזקות אצל הנאמן בתקופת החסימה

6. מימוש ומכירה
   - העובד ממש את האופציות (משלם מחיר מימוש)
   - לאחר תקופת החסימה: מס רווח הון (25%)
   - הנאמן מטפל בניכוי ובדיווח
```

def compare_option_102_tracks(grant_value, exercise_price, sale_price):
    """Compare tax outcomes for Option 102 tracks."""
    gain = sale_price - exercise_price

    capital_gains_track = {
        "track": "Capital Gains (Trustee)",
        "holding_period": "24 months",
        "tax_rate": 0.25,
        "tax_amount": gain * 0.25,
        "net_to_employee": gain * 0.75,
        "employer_deduction": False,
    }

    income_track = {
        "track": "Income (Trustee)",
        "holding_period": "12 months",
        "tax_rate": 0.50,
        "tax_amount": gain * 0.50,
        "net_to_employee": gain * 0.50,
        "employer_deduction": True,
    }

    non_trustee = {
        "track": "Non-Trustee (102)",
        "holding_period": "None",
        "tax_rate": 0.50,
        "tax_amount": gain * 0.50,
        "net_to_employee": gain * 0.50,
        "employer_deduction": True,
    }

    return [capital_gains_track, income_track, non_trustee]

## דוגמאות

### דוגמה 1: רישום סטארטאפ חדש
המשתמש אומר: "אני רוצה לרשום סטארטאפ טכנולוגי חדש בישראל עם השותף שלי"
פעולות:
1. הנחייה לבדיקת שם חברה ב-ica.justice.gov.il
2. המלצה על תקנון סטנדרטי עם הוראות ידידותיות לסטארטאפ
3. חישוב חלוקת מייסדים (למשל 50/50 עם הבשלה של 4 שנים)
4. פירוט שלבים לאחר הרישום (בנק, מס, מע"מ, ביטוח לאומי)
תוצאה: מדריך רישום שלב-אחר-שלב עם טבלת חלוקה.

### דוגמה 2: הגשת מענק לרשות החדשנות
המשתמש אומר: "אנחנו רוצים להגיש בקשה למימון מרשות החדשנות למוצר ה-AI שלנו"
פעולות:
1. הערכת שלב והמלצה על תוכנית (תנופה למוקדם, קרן מו"פ למאוחר)
2. הנחייה לגבי דרישות ההגשה
3. הדגשת מגבלות IP והתחייבויות תמלוגים
4. הנחיות לתבנית תקציב
תוצאה: המלצת תוכנית עם רשימת תיוג להגשה.

### דוגמה 3: הקמת תוכנית אופציות
המשתמש אומר: "אני צריך להקים תוכנית אופציות ל-5 העובדים הראשונים שלי"
פעולות:
1. המלצה על מסלול רווח הון 102
2. הצעת גודל מאגר (10-15% מהחברה)
3. המלצת אפשרויות נאמן
4. פירוט תהליך ההגשה לרשות המסים
5. מתן תנאי הבשלה סטנדרטיים
תוצאה: תוכנית הקמה מלאה של סעיף 102 עם השוואת נאמנים.

## פתרון בעיות

### בעיה: "רשות החדשנות דחתה את הבקשה שלנו"
סיבה: חדשנות טכנולוגית לא מספקת, תוכנית מו"פ חלשה, או בעיות תקציב
פתרון: בקשו משוב מסוקר הרשות, חזקו את מרכיב החדשנות, שקלו הגשה מחדש במחזור הבא. רשות החדשנות מאפשרת הגשה חוזרת.

### בעיה: "תקופת ההחזקה של סעיף 102 לא התקיימה"
סיבה: העובד עזב או המניות נמכרו לפני תום תקופת ההחזקה של 24 חודשים
פתרון: חל הפרש מס -- הרווח ממוסה כהכנסה (עד 50%) במקום כרווח הון (25%). הנאמן ינכה בשיעור הגבוה יותר. תכננו זאת בהסכמי ההעסקה.

### בעיה: "לא ניתן להעביר IP לחו"ל"
סיבה: קניין רוחני שמומן ע"י רשות החדשנות כפוף למגבלות העברה
פתרון: הגישו בקשה לרשות החדשנות לאישור העברה. היו מוכנים לשלם דמי העברה (עד פי 6 מסכום המענק). שקלו מבנה עם חברת בת ישראלית שתחזיק ב-IP.
