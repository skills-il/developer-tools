# מסמכים בעברית

## הנחיות

### שלב 1: הכנת סביבת Python

התקנת python-docx:

```bash
pip install python-docx
```

### שלב 2: הגדרת המסמך ל-RTL עברי

הגדרת כיוון ברירת מחדל ל-RTL עם גופנים עבריים (David, Heebo). שימוש באלמנט `w:bidi` על פסקאות ו-`w:rFonts` עם `w:cs` לגופני Complex Script.

### שלב 3: עיצוב מסמכים ישראלי

- הגדרת עמוד: A4 (21x29.7 ס"מ), שוליים 2.5 ס"מ
- תאריכים: DD/MM/YYYY או שמות חודשים עבריים
- כתובות: רחוב + מספר, עיר, מיקוד (7 ספרות)
- טלפון: פורמט 05X-XXX-XXXX

### שלב 4: תבניות נפוצות

**מכתב רשמי**: תאריך, נמען (לכבוד), נושא (הנדון), פנייה (שלום רב), גוף, סיום (בכבוד רב), חתימה.

**חוזה**: כותרת (ממורכזת), סעיף צדדים, סעיפים ממוספרים, בלוקי חתימה.

### שלב 5: טבלאות RTL

הוספת `w:bidiVisual` למאפייני הטבלה. הפיכת סדר עמודות לתצוגת RTL נכונה.

### שלב 6: מוסכמות טפסים ממשלתיים

- תעודת זהות: פורמט 9 ספרות
- נוסח הצהרה סטנדרטי
- בלוקי חתימה ותאריך

## דוגמאות

### דוגמה 1: יצירת תבנית חוזה בעברית
המשתמש אומר: "צור מסמך Word לחוזה שכירות בעברית"
פעולות:
1. הגדרת מסמך RTL עם גופן Heebo/David
2. יצירת כותרת, פרטי צדדים וסעיפים
3. הוספת פורמט תאריך עברי ומטבע שקלים
4. יצירת קובץ docx. עם יישור פסקאות RTL נכון
תוצאה: חוזה שכירות מקצועי בעברית מוכן לעריכה

### דוגמה 2: יצירת פרוטוקול ישיבה בעברית
המשתמש אומר: "כתוב פרוטוקול ישיבת דירקטוריון מהיום בפורמט Word בעברית"
פעולות:
1. יצירת מסמך Word עם כיוון RTL וכותרת חברה
2. הוספת רשימת משתתפים, נושאי דיון, החלטות
3. עיצוב תאריכים בפורמט DD/MM/YYYY, שעות בפורמט HH:MM
4. הוספת טבלת משימות עם אחראים
תוצאה: פרוטוקול ישיבה מעוצב בעברית

## משאבים מצורפים

### סקריפטים
- `scripts/docx_formatter.py` -- מאמת ומעצב מסמכי Word בעברית לתאימות RTL. הרצה: `python scripts/docx_formatter.py --help`

### חומרי עזר
- `references/israeli-document-formats.md` -- תקני עיצוב מסמכים ישראליים כולל פורמטים של תאריכים, מטבע, כתובות ומבנה מסמכים משפטיים. יש לעיין בו בעת יצירת מסמכים רשמיים או משפטיים בעברית.

## פתרון בעיות

### שגיאה: "טקסט עברי מופיע משמאל לימין"
סיבה: כיוון פסקה במסמך לא הוגדר ל-RTL
פתרון: יש לוודא שלכל פסקה מוגדר `alignment=WD_ALIGN_PARAGRAPH.RIGHT` ושל-run מוגדר `font.complex_script` כ-True עם גופן עברי מתאים.

### שגיאה: "סדר טקסט מעורב עברית/אנגלית שגוי"
סיבה: אלגוריתם BiDi לא מטפל נכון בטקסט דו-כיווני
פתרון: יש לעטוף קטעי אנגלית בסימני LTR (U+200E). לקטעי קוד המשובצים בטקסט עברי, יש להשתמש בגופן monospace עם כיוון LTR מפורש.
